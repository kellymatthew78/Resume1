JavaScript Design Patterns


Closures

Javascript closures are any function that contains an inner function, that references a variable in the parent function. The common implementation is to use the closure pattern to create a private, inaccessible variable environment.

Immediate Functions

Immediate Functions are functions that are invoked as soon as they are defined (ImmediatelyInvoked Function Expression or IIFE). This can also be done for anonymous functions. This paragraph changes color due to an immdiate function!

Module

The module pattern in javascript is a way of creating objects to be used. There are many advanced topics having to do with the module pattern, but the most basic is referred to as ‘revealer’. With the revealer pattern, you’re returning (or revealing) properties or functions that interact with private, unaccessible properties or functions.






Factory

The factory pattern is used for the creation of objects. It allows us to create a generic interface for creating different types of objects. This is very useful for the creation of very complex objects; objects that depend on dynamic factors or application settings.

Observer and Pub Sub

Publisher/subscriber is usually referred to as an observer pattern, but there are other flavors as well. An observer pattern is any in which an object can publish updates to a collection of observing objects. It’s a one-to-many relationship where the dependants of an object are notified of state changes.

Pub/sub is the most common way to implement this pattern. Specifically, pub/sub differs from any other observer pattern in that a third party maintains the publishers and subscribers rather than the publisher itself. It can be easily achieved using jQuery or one of the other dedicated libraries.
